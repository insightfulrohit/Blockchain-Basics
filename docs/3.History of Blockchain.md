## 1) 1991 â€“ Digital Timestamp System (Haber & Stornetta)

This was the first seed of blockchain.

They created a system in which digital documents were timestamped (date/time added) so that the original creation time and later changes could be detected and proven.

---

## Problem

There was no reliable proof of:

- Who created the original file?
- When was it created?
- Whether someone changed the file later or not?

Digital documents cannot be trusted because anyone can copy or modify them.

Even if someone says,

> "This is the original file"

there is zero trust.

---

## Why this system was not enough (limitations)

- It was not a ledger
- There was no network consensus
- There were no incentives/rewards
- Decentralization was missing
- Trust was still dependent on humans/systems, not on a network

---

## Solution

They used cryptographic hashing + timestamping.

---

## Process

- Every document was converted into a hash.
- That hash was stored along with a timestamp.
- If anyone changed the document even slightly,  
  â†’ the hash would change instantly.

This made tampering detectable immediately.
